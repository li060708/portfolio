行业洞察

行业机会

  • 本土平台在数据深度上快速追赶国际巨头
  • AI正在重塑医药数据产品的交互范式（自然语言查询→智能分析）
  • 带量采购、医保谈判等政策催生大量数据分析需求
  • 中小药企的数字化需求远未满足，存在产品下沉空间

行业挑战

  • 医药数据获取壁垒高，源头数据分散且非结构化
  • 国际巨头（IQVIA、Veeva）生态壁垒深厚
  • AI幻觉问题在医药场景容忍度极低
  • 数据合规与隐私保护要求日益严格

Badcase归因方法论

每个Badcase从 Prompt / 知识库 / 模型能力 / 产品设计 四个维度结构化归因

#问题根因关键词
1共情缺失：跳过情绪直接给建议Prompt缺「情绪优先」规则
2绝对化表述：「MCI无法治愈」过度简化知识库缺「部分可逆」平衡表述
3浅层回答：仅5字「有一些用处」缺回答深度检查+追问机制
4兜底僵硬：「我不理解你的问题」异常处理策略单一无引导
5记忆缺失：连续3次重复问同一问题未启用多轮记忆+策略升级
Prompt优化：V1.0 → V2.0

核心思路：角色重塑（信息输出者→温暖同伴） + 流程硬约束（①共情→②重构→③方案→④正向） + 反例禁止

V1.0 Prompt
你是一个专业的MCI健康咨询助手，请基于知识库提供准确的健康建议。回答应保持专业、客观。如果用户的问题你无法回答，请告知用户。
V2.0 Prompt
你是一个温暖而专业的MCI健康咨询助手。

【情绪优先】先识别用户情绪，用1-2句话共情。

【回答框架】①共情确认 → ②认知重构（MCI≠终点） → ③家庭支持方案（患者/照护者/家人分层） → ④正向结尾。

【禁止】不跳情绪直接给建议；不用冰冷术语堆砌；不说「无能为力」。
评估维度V1.0V2.0
共情能力零共情，直接给建议开场共情 + 正常化情绪 + 积极重构
结构化模糊单点建议角色分层（患者/照护者/家庭），可执行
信任感信息输出感强建立力量感，开放式结尾留对话空间
合规未涉及诊断 ✓合规 ✓ + 增加定期随访专业建议
